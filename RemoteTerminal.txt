원격 터미널 접속의 필요성
https://blog.inidog.com/p/201808031462

보통 서버를 구축할 때 관리자와 멀리 떨어져있는 곳에 서버가 위치한 경우가 많음
서버 작업을 처리할 때 마다 해당 서버가 위치한 곳으로 이동할 수는 없고, 각각의
서버로 이동하여 일일히 모니터를 연결하여 작업할 수는 없음 -> 원격 접수는 필수

해당 서버의 주소(IP 등)만 알고 있으면 다른 pc를 통해 바로 접속할 수 있다.
ex) 윈도우의 원격데스크톱연결(RDP)
리눅스는 RDP 프로토콜을 지원하지 않아 다른 프로토콜을 사용해야하고 연결이
가능해지도록 일련의 설명을 해야함
-> 원격접속을 이용하여 터미널 환경을 사용할 수 있도록 고안된 프로토콜이 Rlogin,Telnet,SSH

 Xshell, PuTTY, iTerm이나 SecureCRT는 윈도우에서 원격 리눅스 서버로 터미널 접속 프로토콜을
사용하여 연결할 수 있도록 도와주는 터미널 에뮬레이터 클라이언트.리눅스에서도 마찬가지(가상으로 구현된 복제춤)
 원격지로 연결을 가능하도록 하는 네트워크 접속 도구!

SSH(Secure Shell) : 보안을 중요시한 프로토콜, 최근에는 대부분 ssh를 사용함
Telnet으로 접속하던 ssh로 접속하던 사용하는 환경이 크게 달라지지는 않는다.
(하지만 Telnet은 보안적으로 매우 치명적인 결함이 있음)

*SSH Server 설치하기*

원격 서버에 ssh 프로토콜을 사용하여 터미널에 접속하기 위해 필요한 조건 3가지
- 22 tcp 포트가 방화벽에서 열려있어야함
- ssh 서버 프로그램이 설치 및 구동되고 있어야함
  : 해당 서버가 카페24,고대디 등의 호스팅 서비스나, aws,azure, 구글 클라우드 같은
    클라우드 서비스를 사용 중인 경우 ssh 서비스를 기본 설치된 상태로 제공하기 때문에
    ssh서버 설치 과정을 진행하지 않아고 접속이 가능, 22포트개방도 허용된 상태

    직접 OS를 설치했거나 서버 관리자가 원격 접속을 사용하지 않아서 해당 서버에 SSH
    접속 환경이 구축되지 않았다면 SSH서버 설치를 진행해야 한다.
- ssh 프로토콜로 접속할 수 있는 ssh 클라이언트가 필요

1. 리눅스 최신 배포판들은 SSH서버를 기본적으로 탑재했을 수 있다.
rpm 명령어로  openssh-server를 검색하여 결과가 나온다면 설치된 것

2. ssh 접속 대상이 되는 서버 pc에 직접 로그인 후 , 해당 운영체제의 터미널 열기
   root 권한으로 패키지 관리자를 통해 패키지를 설치 yum install openssh-server
   ,apt install openssh-server

3. ssh 접속을 위한 설정 : ssh 서버를 구동하기 전에 방화벽 프로그램에서 tcp 포트 22번 허용
   ssh 서버는 /etc/sshd/sshd_config라는 설정 파일을 참조한다. ssh 프로토콜 접속 시의
   규칙을 지정하거나 서버의 연결 설정을 정의할 수 있다. 
    - ssh 포트 변경              - 접속 허용 클라이언트 및 패스워드 입력 시도 횟수 제한 설정
     -접속 로그 배너 메세지  - 패스워트, 공개 키 사용자 인증 설정  
   접속만 한다고 가정하면 파일을 특별히 수정할 이유는 없지만 접속에 대한 설정을 변경할 때는 파일의 내용을 수정해줘야함

4. 모든 설정이 완료되면, SSHD 서비스를 시작하여 다른 PC에서 접속할 수 있도록..
    service sshd start <- ssh 서버가 구동중인지 확인할 수 있음 , 결과가 나타나면(active 등) 서비스가 구동중인것임
  
5. ssh 접속 -접속 전 준비 사항
   호스트 주소(IP 또는 도메인-> ifconfig  ip addr 명령어로 확인)와 사용자 정보(Credential)가 있으면 접속 가능
   각각의 ssh 클라이언트 프로그램마다 접속 방법이 다를 수 있다.

6. ssh Client 설치 
   ssh 프로토콜을 사용하여 원격 호스트에 접속하려면 접속 가능한 클라이언트 프로그램이 필요하다.
   리눅스.macOS 운영체제의 경우 기본으로 탑재되어있음
   ssh [UserName]@[HostName] 을 입력하여 연결 가능

   윈도우 운영체제를 사용 중이라면 OpenSSH 클라이언트가 따로 없기 때문에 OpenSSH를 이용한
   외부 터미널 클라이언트를 설치해야 한다.
   ex)  PuTTY나 Xshell, SecureCRT, TeraTerm 그리고 MobaXterm 

7-1. ssh 클라이언트에서 접속 - PuTTY 
   Session 선택 -> 호스트 주소 입력 -> ssh 선택 -> open 클릭하여 연결
   ----> 호스트 키 검증 창 뜨면 예를 클릭하여 현재 pc에 서버 pc의 호스트 키를 저장하여 추후 동일 서버 여부를 체크함
   PuTTY 창에서 사용자이름과 암호를 입력하면 원격 터미널 접속에 성공
  
7-2. ssh 클라이언트에서 접속 - Xshell
   xshell을 실행 -> 새로운 세션 생성(파일-새로만들기) -> 연결 선택 -> 세션이름 지정
   -> ssh 선택 -> 호스트 주소 입력 -> 연결 클릭 하여 파일 저장 후 바로 연결
   ----> 호스트 키 검증 창 뜨면 수락 및 저장을 틀릭하여 호스트 키를 저장
   ssh 사용자 이름 창이 나타나면 사용자 계정 이름 입력, ssh 사용자 인증 창이 나타나면 암호를 입력하여 인증을 완료
